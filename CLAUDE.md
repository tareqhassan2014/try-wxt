## Overview

Browser extension built with [WXT](https://wxt.dev) + React 19 + TypeScript. WXT is a Vite-based framework that handles manifest generation, multi-browser builds (Chrome/Firefox), HMR, and the dev harness.

## Commands

Package manager is **pnpm** (pnpm 10+).

- `pnpm dev` — run dev server (Chrome) with HMR, auto-loads extension
- `pnpm dev:firefox` — dev server targeting Firefox
- `pnpm build` / `pnpm build:firefox` — production build to `.output/`
- `pnpm zip` / `pnpm zip:firefox` — package extension for store submission
- `pnpm compile` — type-check only (`tsc --noEmit`), no build

No test runner or linter configured. `compile` is the only static check.

## Architecture

WXT uses **convention over configuration** — files in `entrypoints/` become extension entry points by filename; no manual manifest wiring. `wxt.config.ts` only registers `@wxt-dev/module-react`.

Entrypoints (each maps to a manifest surface):
- `entrypoints/background.ts` — service worker; export via `defineBackground()`
- `entrypoints/content.ts` — content script; export via `defineContentScript({ matches })`. Current match: `*://*.google.com/*`
- `entrypoints/popup/` — React popup UI (`index.html` → `main.tsx` → `App.tsx`)

WXT injects globals — `defineBackground`, `defineContentScript`, `browser` (cross-browser WebExtension API) are auto-imported, no import statements needed.

Path aliases: `@/` → project root (e.g. `@/assets/react.svg`); `/file` → `public/` (e.g. `/wxt.svg`).

## Generated / do-not-edit

- `.wxt/` — generated types and tsconfig base (`tsconfig.json` extends `./.wxt/tsconfig.json`). Regenerated by `wxt prepare` (runs on `postinstall`).
- `.output/` — build artifacts.

## pnpm build scripts

Dependency build scripts are gated in `pnpm-workspace.yaml` under `allowBuilds` (pnpm 10+ blocks them by default). `esbuild` and `spawn-sync` are explicitly allowed there. Configure build-script policy in `pnpm-workspace.yaml`, NOT the deprecated `pnpm` field in `package.json`.
